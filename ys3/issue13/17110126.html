Notes

If you're serious about Z80 programming, there's a few little things that'll really help you out. First things first, you really need a printed table of Z80 instructions. I know, it's a pain to print it out, but the benefits in terms of ease of use and time-saving make it worthwhile. You can find one of the lists floating around at http://www.ticalc.org/ here:here.

Next, an integrated development environment will make things much, much easier. I use Assembly Studio 86; it runs under Windows, and with a few options set I've got one-click conversion from object code to .TAP file (by way of the Infamous Blood's BINTAP), which saves all that tedious messing about with command lines.

Now, it's also extremely desirable to get a decent emulator. Now, many of you out there already have your favoured emulators (X128 springs to mind, as does Realspec, the Spectrum Emulator of Champions), but when I'm programming, the only emulator I even considered was ZX32. It runs in a window, which saves my monitor from clicking back and forth between resolutions every minute or so, and also allows me to look at my code while the program's running. The only other Windows-based emulator I can think of is Gerton Lunter's Z80, which is also rather nice, but costs money.

If you're not running Windows, there are bound to be similar solutions. Unfortunately I'm unaware of all of them. If you've got other ways to make Z80 development easier, do share them with me, so I can add your advice to this section in future.
